This is a toolbox for the software product client: the person with a vision of a product, and a team of developers to help build it.
There are many resources for agile developers. They focus on how programmers can radically alter the way they code, to deliver value to clients faster.
There are many resources for agile project managers. They focus on how project managers can help manage risks and remove obstacles in an agile development project.
This resource is for agile clients. It focuses on how to apply agile values and principles to product design, from initial idea formation to a testable product.
The tools in this toolbox are ideas. Most of these ideas are simple practices that agile team have found useful the past 20 years for achieving the primary goal of agile: delivering value quickly and frequently.
Also in the toolbox are some common activities that clients will engage in frequently with their developers. These activities are where you work with the developers to make your vision real... and realistic.
Agile is not about using these tools and doing these activities. You could do all of these things and not be agile. You could do none of them and still be agile. Agile is a mindset that constantly looks for ways in which more value can be sustainably delivered more frequently.
Warning: Agile development is a contact sport. Being an agile client requires frequent, effective communication with developers, not about code or technology, but about end user product value.
For those just beginning the agile journey, here are some key starting points:

  • Effective user testing for each slice
  • Planning the next slice
  • Estimating and tracking
