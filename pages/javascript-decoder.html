Arrow functions

In React code, you'll often see functions defined using arrow functions

const square = x => x * x;

rather than function notation

function square (x) { return x * x; }

This is a shorter simpler syntax for functions that just return the value of an expression.

Unlike function, arrow functions do not re-bind the special variable this.

Method definitions

In React code, you'll often see methods for objects and class components defined like this:

const obj = {
  foo() {
    return 'bar';
  }
}

rather than like this:

const obj = {
  foo: function foo() {
    return 'bar';
  }
}
Destructuring

You'll often see braces around React function parameters, like this 

const Hello = ({name}) => <h1>Hello, {name}</h1>;

This is using destructuring assignment. It is equivalent to writing this

const Hello = (props) => {
  const name = props.name;
  <h1>Hello, {name}</h1>;
}

Destructuring is also used with import statements:

import { useState } from 'react';

Destructuring is used with arrays:

const [state, setState] = useState(initialState);
React.Component

React components are often defined like this: 

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

This defines the component Welcome as a JavaScript subclass. The class definition can include a constructor, local state variables, and React lifecycle methods. See React class components for more information.

Functional Components

More recent React code may define components as simple functions: 

const Welcome = (props) => <h1>Hello, {props.name}</h1>;

This defines the component Welcome as a JavaScript function. It is treated as the render() function for the component. There is no local state, constructor, or lifecycle methods.
