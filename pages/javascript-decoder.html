Arrow functions

In React code, you'll often see functions defined using arrow functions

const square = x => x * x;

rather than function notation

function square (x) { return x * x; }

This is a shorter simpler syntax for functions that just return the value of an expression.

Unlike function, arrow functions do not re-bind the special variable this.

Method definitions in objects

In React code, you'll often see methods for objects and class components defined like this:

const obj = {
  foo() {
    return 'bar';
  }
}

rather than like this:

const obj = {
  foo: function foo() {
    return 'bar';
  }
}
Destructuring assignment

You'll often see braces around React function parameters, like this 

const Hello = ({name}) => <h1>Hello, {name}</h1>;

This is using destructuring assignment. It is equivalent to writing this

const Hello = (props) => {
  const name = props.name;
  <h1>Hello, {name}</h1>;
}

Destructuring is also used with import statements:

import { useState } from 'react';

Destructuring is used with arrays:

const [state, setState] = useState(initialState);
React.Component subclasses

React components are often defined like this: 

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

This defines the component Welcome as a JavaScript subclass. The class definition can include a constructor, local state variables, and React lifecycle methods. See React class components for more information.

Functional Components

More recent React code may define components as simple functions: 

const Welcome = (props) => <h1>Hello, {props.name}</h1>;

This defines the component Welcome as a JavaScript function. It is treated as the render() function for the component. There is no local state, constructor, or lifecycle methods.

Export and import

Definitions marked with export, e.g.,

export const loadCart = (cartProducts) => ...

can be imported by name for use in another file, e.g.,

Export default and import

Definitions marked withexport default, e.g.,

export default class Welcome extends React.Component {
  ...
}

can be imported by name for use in another file, e.g.,

import Welcome from './Welcome'
const declarations

Most local variables in React code are declared with const:

const weekdays = ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun'];
weekdays = ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat'];  // illegal

A const variable, when applicable, is preferred to a var variable, because it is block-scoped, rather than function-scoped, and can't be reassigned another the value. These restrictions make code with const variables easier to maintain than code with var variables.

Note that you can modify a structure stored in a const variable, e.g., 

weekdays.unshift(weekdays.pop()) // weekdays is now ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat']
let declarations

React code that needs local variables that can be reassigned are declared with let:

let weekdays = ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun'];
        weekdays = ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat'];  // OK

let is preferred to var because it is block-scoped.
