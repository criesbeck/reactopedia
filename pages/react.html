React is a JavaScript framework for building dynamic web apps. Everything is defined in terms of components. A component is entered onto a page like an HTML element, e.g., <Product>, but underneath is JavaScript code that contains data and instructions how to convert that data into actual HTML.
Understanding React requires understanding a large ecosystem of technologies.
First is modern JavaScript, more specifically EcmaScript 2015 and later. EcmaScript 2015 is sometimes referred to as ES6.
Second is NodeJS, or just Node. This is a way to run JavaScript as a standalone system, like Python, rather than in the browser. Node is used as a scripting language to implement the React command line interfaces (CLI).
A critical part of Node is npm, the Node Package Manager, that is used to download libraries and run scripts to build applications and start web servers.
React programs get complicated fast. To debug them, use the Google Chrome debugger. Because React code is combined and transformed in many ways to become a normal HTML page, you need to add the React extension to Chrome.
React components should be as independent as possible. To share data between components, you use a library called Redux.
Most modern web apps are single page web apps. That means that there are multiple user screens, but only one actual web page. JavaScript code makes it look like you are moving around different pages. The library that handles this is the React Router.
