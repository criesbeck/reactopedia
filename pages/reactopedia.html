This is a centralized resource for students learning React and the React ecosystem.
Understanding React requires understanding a large ecosystem of technologies.
To understand and write React code, you need to be familiar with modern JavaScript, i.e., EcmaScript 2015 (also known as ES6) and later.
To create and run React apps, you need NodeJS (also known as just Node). This is a standalone system for running scripts, similar to Python, but based on JavaScript. Node is used to implement the React command line interfaces (CLI).
A critical part of Node is npm (the Node Package Manager). npm downloads libraries and run scripts to build applications and start web servers.
Of course there is React itself. This is a library of code for writing interactive web applications in JavaScript.
React programs get complicated fast. To debug them, use the Google Chrome debugger. Because React code is combined and transformed in many ways to become a normal HTML page, you need to add the React extension to Chrome.
Most React apps are single page web apps. There is one web page that acts like a multi-page application, with distinct URLs for each page. The standard JavaScript library for doing this is ReactRouter.
React components should be as independent as possible, but most applications have common state, i.e., data, that needs to be shared between components. The most commonly used library to do that is Redux.
